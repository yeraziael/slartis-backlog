# P0: Coding Result Gate - PR, CI, Review und DoD

> *This was generated by AI during triage.*

Parent: #149. Abhaengigkeit: #156.

## Ziel

Nach dem Vekling-Dispatch wird ein Coding-Ergebnis nicht mit erfolgreicher Annahme gleichgesetzt. Branch/Commit, Pull Request, CI, Review und Definition-of-Done bilden ein nachvollziehbares Gate vor jeder autorisierten Merge-Entscheidung.

## Scope

- Vekling-Ergebnis auf Branch, Commit und Artefakte validieren.
- Pull Request deterministisch erstellen oder einen bestehenden idempotent wiederverwenden.
- CI-Status und maschinenlesbaren Report auswerten.
- Review-Ergebnis und DoD getrennt vom Worker-Ergebnis persistieren.
- Fehler, fehlende Artefakte und abgelehnte Reviews eskalieren.
- Kein Auto-Merge und keine Rechteausweitung.

## Tests zuerst

- Erfolgreiches Worker-Ergebnis erzeugt genau einen PR.
- Wiederholung erzeugt keinen zweiten PR.
- Fehlende Commits, rote CI oder abgelehntes Review verhindern die Freigabe.
- Tests laufen ohne produktive Repositories oder Secrets.

## Definition of Done

- Vollstaendiger Contract-Lifecycle von Vekling-Ergebnis bis freigabefaehigem PR.
- Lokale und Gitea-CI gruen.
- Merge bleibt eine separat autorisierte Aktion.
