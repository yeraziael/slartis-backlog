# Playwright — Contracts

Normative keywords per RFC 2119.

## Synthetic Test Identities Contract

### Identity Specification

| Field | Requirement |
|---|---|
| Username | `pw-e2e-<service>-<role>` (e.g., `pw-e2e-abs-user`, `pw-e2e-abs-admin`, `pw-e2e-abs-denied`) |
| Display name | Same as username — visibly marks the account as automation |
| Email | `pw-e2e-<service>-<role>@homelab.test` |
| Personal data | NONE — no real name, no real email, no real address |
| Password | Generated by Keycloak, injected at runtime |
| Group membership | Minimum required for role: user group, admin group, or none (denied) |
| Rotation | Password rotatable without test code changes |

### Obligations

- The Keycloak administrator **MUST** create and maintain synthetic identities.
- The runner **MUST** receive credentials via environment variables, never from code or config.
- Test suites **MUST NOT** hardcode usernames or passwords.
- Evidence **MUST** be sanitised to remove any credential exposure before publication.

## Evidence Manifest Contract

### Minimum Fields

```json
{
  "manifest_version": 1,
  "source_commit": "<sha256>",
  "architecture_commit": "<sha256>",
  "service_name": "audiobookshelf",
  "service_version": "<image-digest-or-version>",
  "runner_image": "mcr.microsoft.com/playwright:v1.52.0@sha256:...",
  "environment": "production",
  "suite": "smoke",
  "result": "pass",
  "duration_seconds": 42,
  "timestamp": "2026-07-22T15:00:00Z",
  "tests": {
    "total": 10,
    "passed": 10,
    "failed": 0,
    "prerequisite_errors": 0,
    "skipped": 0
  }
}
```

### Bundle Structure

```
evidence-<service>-<suite>-<timestamp>/
├── manifest.json
├── results.json
├── junit.xml
├── screenshots/
│   └── <test-name>-<timestamp>.png
├── traces/
│   └── <test-name>-<timestamp>.zip
└── videos/
    └── <test-name>-<timestamp>.webm
```

### Obligations

- The runner **MUST** produce `manifest.json` for every run.
- The runner **MUST** include source and architecture commit SHAs.
- The runner **MUST** include the service version or image digest.
- Evidence **MUST** be suitable for inclusion in ACP review packages.

## Screenshot Contract

| Condition | Action |
|---|---|
| Test fails | Screenshot **MUST** be captured and retained |
| Test passes | Screenshot **MAY** be captured if explicitly configured |
| Screenshot contains session | Runner **SHOULD** verify no tokens visible in captured area |
| Retention | Screenshots **SHOULD** be retained until review is complete |

## Trace Contract

| Condition | Action |
|---|---|
| Test fails | Playwright trace **MUST** be retained |
| Test passes | Trace **SHOULD** be suppressed |
| Trace contains credentials | Runner **MUST** sanitise before storage |
| Retention | Traces **SHOULD** have a configurable expiry (default: 30 days) |

## Video Contract

| Condition | Action |
|---|---|
| Test fails | Video **MAY** be retained if configured |
| Test passes | Video **SHOULD** be suppressed |
| Retention | Videos **SHOULD** follow trace retention policy |

## Browser Prerequisites Contract

| Prerequisite | Check | On Failure |
|---|---|---|
| Service HTTPS reachable | `GET /` → HTTP 200 | `prerequisite_error` |
| Service healthcheck passes | Service-specific health endpoint | `prerequisite_error` |
| Keycloak reachable | OIDC discovery or login page | `prerequisite_error` |
| Synthetic identity exists | Login flow succeeds | `prerequisite_error` |
| Controlled test data accessible | Service API or UI check | `prerequisite_error` |
| DNS resolution | Resolve service FQDN | `prerequisite_error` |
| TLS certificate valid | `node -e tls.check` or Playwright check | `prerequisite_error` |

## Service Onboarding Contract

### Service Responsibilities

The adopting service **MUST**:

1. Document required user journeys and acceptance criteria.
2. Provide controlled test data in a dedicated test library or test area.
3. Document why browser-level testing adds value beyond unit, API and integration tests.
4. Define smoke, regression and E2E coverage expectations.
5. Provide stable selectors (prefer `data-testid`) or document alternatives.

### Platform Responsibilities

The platform **MUST**:

1. Provide shared OIDC login fixture.
2. Provide synthetic user-role fixture.
3. Provide evidence manifest generation.
4. Provide console-error capture helper.
5. Document execution conventions.
6. Provide runner definition (pinned container image, configuration).

### Obligations

- Onboarding **MUST NOT** require changes to shared platform fixtures for common patterns.
- Service-specific page objects **MUST** be implemented in the service suite directory.
- Service suites **MUST** use platform-provided fixtures for authentication and evidence.

## Deterministic Execution Contract

| Aspect | Requirement |
|---|---|
| Browser context | Fresh context per test (isolation) |
| Test data | Deterministic — same input, same output |
| Timing | `waitFor` and timeouts with configurable defaults |
| Screenshots | Pixel-identical on same browser version and viewport |
| Exit codes | 0 = pass, 1 = test failure, 2 = prerequisite error |
| Randomness | Seeded random if any randomness in test flows |

## Failure Semantics Contract

| Result | Meaning | Actions |
|---|---|---|
| `pass` | All assertions passed | No further action |
| `fail` | One or more assertions failed | Screenshot + trace retained, exit code 1 |
| `prerequisite_error` | Precondition not met (DNS, TLS, Keycloak, health) | Screenshot + trace retained, exit code 2 |
| `skipped` | Test explicitly skipped (e.g., feature not available) | Reported in results, not a failure |
| `error` | Test threw unexpected exception | Screenshot + trace retained, exit code 1 |

A suite with any `prerequisite_error` results **MUST NOT** be reported as a product regression.
