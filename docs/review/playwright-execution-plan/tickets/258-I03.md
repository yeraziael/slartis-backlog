## Normative Implementation Contract

- Exit 0 means pass; exit 1 means test failure or unexpected runner error; exit 2 means prerequisite error.
- A prerequisite error MUST NOT be labelled as product regression.
- Result selection MUST follow PW-D02 and be unit-testable without a live service.

## Acceptance Criteria

- All forced outcomes emit one stable machine-readable result and exact exit code.
- Mixed-result fixture follows documented precedence.
- Unknown exceptions cannot produce exit 0 or 2.

## Required Tests

- `make test-playwright-platform` with all result fixtures
- Invoke each forced outcome and assert shell exit status 0, 1 or 2.

## Required CI Validation

- Required platform CI executes all result-semantic fixtures, including non-zero expectations inside a harness that itself exits zero.
- From the repository root, run `make lint`, `make test`, and `git diff --check origin/main...HEAD`.
- The required Gitea Actions jobs for the PR head MUST be green. An absent job is an evidence deficiency, not a pass.
