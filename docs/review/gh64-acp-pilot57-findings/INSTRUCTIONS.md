# ACP Specification at Review Head

The ACP specification at the time of this review is `0.1.0-draft`. The
normative content (`SPEC/`) is effectively empty — only a `.gitkeep` file.

## Review Convention

- Findings are classified into four categories: **Implementation**, **Prompt**,
  **Reviewer**, **ACP**.
- Each finding records whether it is a candidate for a future ACP revision.
- The findings primer in `FINDINGS/README.md` defines the workflow and
  register.
- `FINDINGS/TEMPLATE.md` provides a reusable template for future pilots.
- `DECISIONS/README.md` captures architectural decisions derived from findings.

## Review Boundary

Changes in this PR are **documentation only**. No change to:
- ACP specification (SPEC/)
- Schemas (SCHEMAS/)
- Release version (VERSION)
- Conformance tests (CONFORMANCE/)
